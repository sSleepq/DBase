# Функция TRUNCATE в SQL

## Описание

Функция **TRUNCATE** округляет число до определённого знака, **отбрасывая лишние цифры** без применения правил математического округления.


В отличие от `ROUND`:
- `TRUNCATE(100.339, 2)` → `100.33` (просто отсекает «9»);
- `ROUND(100.339, 2)` → `100.34` (округляет по правилам: 9 > 5 → увеличивает предыдущий знак).

## Синтаксис

```sql
SELECT TRUNCATE(поле, сколько_знаков_оставить) FROM имя_таблицы WHERE условие;
```

- `поле` — столбец с числовыми данными;
- `сколько_знаков_оставить` — количество знаков после запятой, которые нужно сохранить (целое число).

## Пример таблицы

Таблица `employees`:

| id (айди) | name (имя) | age (возраст) | salary (зарплата) |
|-----------|-----------|-------------|--------------|
| 1 | user1 | 23 | 100.339 |
| 2 | user2 | 24 | 100.331 |

## Пример использования

**Задача:** усечь зарплату до двух знаков после запятой.

**SQL‑запрос:**
```sql
SELECT *, TRUNCATE(salary, 2) AS salary FROM employees;
```

**Результат:**

| id (айди) | name (имя) | age (возраст) | salary (зарплата) |
|-----------|-----------|-------------|--------------|
| 1 | user1 | 23 | 100.33 |
| 2 | user2 | 24 | 100.33 |

## Важные замечания

1. **Принцип работы:**  
   - Функция отсекает цифры после указанного знака, не анализируя следующую цифру.  
   - Например: `TRUNCATE(100.999, 1)` → `100.9` (не округляет до `101.0`).


2. **Отрицательные числа:**  
   - Работает аналогично: `TRUNCATE(-100.339, 2)` → `-100.33`.


3. **Количество знаков:**  
   - Если указать `0`, число усекается до целого: `TRUNCATE(100.339, 0)` → `100`.  
   - Если указать отрицательное число (например, `-1`), усечение происходит до десятков: `TRUNCATE(123.45, -1)` → `120`.


4. **Типы данных:**  
   - Работает с числовыми типами (DECIMAL, FLOAT, DOUBLE и др.).


5. **Отличие от CAST/CONVERT:**  
   - `TRUNCATE` не меняет тип данных, а лишь модифицирует значение.  
   - Например, результат остаётся числом с плавающей точкой, а не превращается в строку.


6. **Поддержка в СУБД:**  
   - Функция `TRUNCATE` есть в MySQL.  
   - В PostgreSQL и SQL Server аналогичную логику можно реализовать через `TRUNC()` или `ROUND()` с дополнительными манипуляциями.
