# Функция SUM в SQL

## Описание

Функция **SUM** вычисляет сумму значений указанного числового поля по всем выбранным строкам.  

Например, если в таблице есть поле `salary` (зарплата), с помощью `SUM` можно найти:
- общую сумму зарплат всех сотрудников;
- сумму зарплат по определённому отделу или возрастной группе (с использованием `WHERE`).

## Синтаксис

```sql
SELECT SUM(поле) FROM имя_таблицы WHERE условие;
```

**Параметры:**
- `поле` — имя числового столбца, значения которого нужно просуммировать;
- `имя_таблицы` — название таблицы с данными;
- `условие` (опционально) — условие фильтрации строк перед суммированием.

## Пример таблицы

Таблица `employees`:

| id (айди) | name (имя) | age (возраст) | salary (зарплата) |
|-----------|-----------|-------------|--------------|
| 1 | user1 | 23 | 100 |
| 2 | user2 | 24 | 200 |
| 3 | user3 | 25 | 300 |

## Примеры использования

### Пример 1. Суммирование всех зарплат

**Задача:** найти суммарную зарплату всех работников.

**SQL‑запрос:**
```sql
SELECT SUM(salary) AS sum FROM employees;
```

**Результат:**

| sum (суммарная зарплата) |
|-----------------------|
| 600 |

**Пояснение:**  
$100 + 200 + 300 = 600$

### Пример 2. Суммирование с условием

**Задача:** найти суммарную зарплату работников, у которых `id` меньше 3.

**SQL‑запрос:**
```sql
SELECT SUM(salary) AS sum FROM employees WHERE id < 3;
```

**Результат:**

| sum (суммарная зарплата) |
|-----------------------|
| 300 |

**Пояснение:**  
Учитываются только строки с `id = 1` и `id = 2`:  
$100 + 200 = 300$


## Важные замечания

1. **Тип данных:**  
   - `SUM` работает только с числовыми полями (INT, DECIMAL, FLOAT и т. п.).
   - Если поле содержит `NULL`, эти значения игнорируются (не влияют на сумму).


2. **Пустой результат:**  
   - Если ни одна строка не соответствует условию `WHERE`, результат будет `NULL`.


3. **Псевдонимы (AS):**  
   - Использование `AS sum` позволяет задать удобное имя для результирующего столбца.


4. **Сочетание с GROUP BY:**  
   - `SUM` часто используется с `GROUP BY` для подсчёта сумм по группам (например, сумма зарплат по отделам).


5. **Производительность:**  
   - Для больших таблиц рекомендуется использовать индексы по полям, указанным в `WHERE`, чтобы ускорить выполнение запроса.
