# Создание таблиц в SQL

Для создания таблиц применяется команда `CREATE TABLE`. С этой командой можно использовать:
- операторы, определяющие столбцы таблицы и их атрибуты;
- операторы, задающие свойства таблицы в целом.

Одна база данных может содержать до 2 млрд таблиц.

## Общий синтаксис создания таблицы

```sql
CREATE TABLE название_таблицы
(
    название_столбца1 тип_данных атрибуты_столбца1,
    название_столбца2 тип_данных атрибуты_столбца2,
    ...
    название_столбцаN тип_данных атрибуты_столбцаN,
    атрибуты_таблицы
)
```

## Правила именования таблиц

Имя таблицы — её идентификатор в базе данных, поэтому оно должно быть **уникальным** и соответствовать следующим требованиям:
- длина — не более 128 символов;
- допустимые символы: алфавитно‑цифровые, `$` и `_`;
- первый символ — буква или `_`;
- запрещены пробелы и ключевые слова языка Transact‑SQL.

### Особые случаи
- Если идентификатор содержит пробельные символы, его нужно заключать в **кавычки**.
- Если необходимо использовать ключевое слово в качестве имени, его помещают в **квадратные скобки**.

### Примеры корректных идентификаторов
- `Users`
- `tags$345`
- `users_accounts`
- `"users accounts"`
- `[Table]`

## Структура определения таблицы

После имени таблицы в скобках указываются:
1. Параметры всех столбцов.
2. В конце — атрибуты, относящиеся ко всей таблице.

**Важно:** атрибуты столбцов и таблицы — необязательные компоненты.

## Минимальные требования к команде `CREATE TABLE`

В простейшем виде команда должна содержать:
- имя таблицы;
- имена столбцов;
- типы данных столбцов.

## Ограничения и требования к столбцам

- Количество столбцов: от 1 до 1024.
- Каждое имя столбца должно быть уникальным в рамках таблицы.
- Каждому столбцу должен быть назначен тип данных.

## Пример: создание таблицы `Customers`

```sql
CREATE TABLE Customers
(
    Id INT,
    Age INT,
    FirstName VARCHAR(20),
    LastName VARCHAR(20),
    Email VARCHAR(30),
    Phone VARCHAR(20)
)
```
