# Лабораторная работа: Разработка базы данных «Банк»

## Цель работы
Освоить принципы проектирования реляционных баз данных: создание структуры таблиц, определение ключевых полей, установление связей между таблицами.

## Теоретическая часть

Реляционная база данных — это набор взаимосвязанных таблиц, где каждая таблица представляет собой отдельный объект предметной области. Важнейшие элементы проектирования:
* **ключевые поля** — обеспечивают уникальность записей и служат для связывания таблиц;
* **связи между таблицами** — устанавливают логические отношения между данными разных таблиц.

## Задание

Разработать базу данных «Банк», состоящую из трёх взаимосвязанных таблиц.

### 1. Создание структуры таблиц

#### Таблица «Клиенты»

**Назначение:** хранение персональных данных клиентов банка.

| Поле | Тип данных | Свойства 
|------|-----------|---------|
| КодКлиента | Числовой (счётчик) | **Первичный ключ** | Автоматически генерируется |
| Фамилия | Текстовый | Обязательное | |
| Имя | Текстовый | Обязательное | |
| Отчество | Текстовый | Необязательное | |
| Паспорт | Текстовый | Обязательное, уникальный | Серия и номер |
| Телефон | Текстовый | Обязательное | Формат +X (XXX) XXX-XX-XX |
| Адрес | Текстовый | Обязательное | Полный адрес проживания |
| ЗаработнаяПлата | Денежный | Обязательное | Ежемесячный доход |

#### Таблица «ВидыКредитов»

**Назначение:** описание кредитных продуктов банка.

| Поле | Тип данных | Свойства 
|------|-----------|---------|
| КодКредита | Числовой (счётчик) | **Первичный ключ** | Автоматически генерируется |
| НазваниеКредита | Текстовый | Обязательное | Например: «Ипотека», «Автокредит» |
| ПроцентнаяСтавка | Числовой | Обязательное | Годовая ставка в % |
| УсловияПредоставления | Текстовый | Необязательное | Особые требования к заёмщику |

#### Таблица «ПредоставленныеКредиты»

**Назначение:** учёт выданных кредитов.

| Поле | Тип данных | Свойства 
|------|-----------|---------|
| НомерПП | Числовой (счётчик) | **Первичный ключ** | Порядковый номер записи |
| КодКлиента | Числовой | Внешний ключ | Ссылка на таблицу «Клиенты» |
| КодКредита | Числовой | Внешний ключ | Ссылка на таблицу «ВидыКредитов» |
| ДатаПредоставления | Дата/время | Обязательное | Дата выдачи кредита |
| Срок | Числовой | Обязательное | Срок кредитования в месяцах |
| ДатаВозврата | Дата/время | Необязательное | Планируемая дата погашения |
| Сумма | Числовой | Обязательное | Сумма выданного кредита |
| ОтметкаОВозврате | Логический | Обязательное | `Да`/`Нет` (погашен ли кредит) |

### 2. Установление связей между таблицами

Необходимо создать **два отношения типа «один‑ко‑многим»**:

1. **Связь «Клиенты → ПредоставленныеКредиты»**  
   * **Главная таблица:** «Клиенты» (сторона «один»).  
   * **Подчиненная таблица:** «ПредоставленныеКредиты» (сторона «много»).  
   * **Связующее поле:** `КодКлиента`.  
   * **Свойства связи:**  
     * Обеспечение целостности данных: **включено**.  
     * Каскадное обновление связанных полей: **включено**.  
     * Каскадное удаление связанных записей: **включено**.

2. **Связь «ВидыКредитов → ПредоставленныеКредиты»**  
   * **Главная таблица:** «ВидыКредитов» (сторона «один»).  
   * **Подчиненная таблица:** «ПредоставленныеКредиты» (сторона «много»).  
   * **Связующее поле:** `КодКредита`.  
   * **Свойства связи:**  
     * Обеспечение целостности данных: **включено**.  
     * Каскадное обновление связанных полей: **включено**.  
     * Каскадное удаление связанных записей: **включено**.
